Nimbus

A visual autonomy environment for connected flight systems.

Nimbus brings live vision, depth, radio control, telemetry, and programmable AI pipelines into one development workspace.

C++Computer visionELRS / CRSFONNX
VisionDepthControlNimbus

02A Perception, decision, and control joined in one live workspace.

01 The problem

Autonomy work breaks at the seams.

Computer vision often lives in one tool, radio control in another, and telemetry somewhere else. Nimbus treats the flight system as one feedback loop so every signal can be inspected, connected, and tested without rebuilding the surrounding environment.

The result is a development surface for exploring intelligent control before the pieces harden into a production stack.

02 System capabilities
01

See in real time

Live FPV video, YOLO detection, tracking, depth estimation, and overlays share the same operational view.

02

Program visually

Reusable skill blocks can be connected into typed pipelines, configured live, and validated before execution.

03

Control the vehicle

CRSF and ExpressLRS integration connects RC channels, telemetry, link health, and software-driven overrides.

04

Measure the loop

Latency, update rates, system performance, radio quality, and flight data stay visible while the system runs.

05

Bring custom models

ONNX support makes it possible to test custom perception models without replacing the surrounding application.

06

Iterate on hardware

The interface is designed around real devices, real links, and the constraints that appear outside simulation.

Research status

Working prototype.
Private repository.
