Forge

A model-agnostic AI design studio.

Forge turns a design brief into working interface directions. Compare concepts, change the details, and export the selected implementation without locking the workflow to one model provider.

View on GitHub ↗ Quick start

02A A design direction moves from conversation to a multi-file React project.

01 Why it exists

Design with the best model for each step.

One model can be strong at visual direction. Another can be better at full implementations or focused edits. Forge separates those jobs so the user can choose the provider and model tier for each phase.

The interface keeps the brief, design controls, generated files, versions, and comments in one workspace. The result is a design process that stays visual while it produces real code.

02 The workflow
01

Describe the outcome

Start with a plain-language brief. Forge keeps the design request beside the live work.

02

Generate directions

Ask the design model for distinct concepts, then compare the visual and structural choices.

03

Choose the models

Use OpenAI or Anthropic for design, code, and edit work. Model IDs remain configurable.

04

Adjust the system

Change color, type, density, and other design controls without losing the selected direction.

05

Comment in context

Select an interface element and attach a focused note to the part that must change.

06

Export working files

Package the implementation, design tokens, component data, and handoff instructions.

03 Product view
04 Model routing

One workflow.
Many model choices.

Forge routes each request through the OpenAI Responses API or the Anthropic Messages API. The selected provider can change by work phase without a source-code change.
